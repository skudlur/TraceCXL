# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.0] - TraceCXL Initial Release

### Added
- **Flit-Level Serialization**: Messages are now properly broken down into 64B Flow Control Units (flits) to accurately model serialization delay and link-layer interleaving.
- **Credit-Based Flow Control**: Implemented strict link-level `tx_credits` between Hosts and Switches to ensure lossless transmission across the fabric.
- **Virtual Channels (QoS)**: Traffic is separated into 4 distinct QoS priority levels (LOW, MEDIUM, HIGH, CRITICAL). Each priority level is backed by dedicated egress queues and independent Virtual Channel credits.
- **Adaptive Rate Control (ECN)**: Implemented Explicit Congestion Notification (ECN). Switches probabilistically mark packets when queues exceed thresholds, and Hosts respond with AIMD (Additive-Increase/Multiplicative-Decrease) throttling.
- **Trace Replay Engine**: Added `TraceReplayWorkload` to ingest deterministic CSV memory traces generated by external profilers (e.g., QEMU, Intel PIN, CXLMemSim).
- **Trace Generation and Export**: Added `export_trace` to allow researchers to dynamically generate synthetic statistical traces with randomized read/write ratios and export them for offline use.
- **Semantic Validation Engine**: Integrated a passive `ProtocolValidator` that hooks into the discrete-event engine to enforce strict CXL.mem transaction invariants (e.g., 64-byte alignment, exact 64B cacheline payloads).
- **Realistic Application Profiles**: Added `examples/application_profiles.py` demonstrating how to model real-world memory access patterns like LLM Inference (bandwidth-bound) and Memcached/Redis (congestion-bound hotspot).
- **Topology Support**: Out-of-the-box support for Single Switch and Two-Tier (Spine-Leaf) datacenter topologies with configurable buffering.
- **Metrics and Analysis**: Integrated `MetricsCollector` to track end-to-end latency, queue depths, and ECN markings, coupled with Plotly graphing utilities (`plot_latency_cdf`, `plot_queue_occupancy_heatmap`).
- **Rename**: Officially rebranded the repository to `TraceCXL` to emphasize its trace-driven capabilities and differentiate it from heavier software execution simulators.
